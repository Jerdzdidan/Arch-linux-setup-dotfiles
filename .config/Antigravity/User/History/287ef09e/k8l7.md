# Walkthrough: Student Email Detection & Prompt Modal

## Summary

Added a feature that detects whether a logged-in student has an email address. If they don't, an **un-dismissable Bootstrap modal** prompts them to enter one. The email is saved via AJAX and persisted to the `users` table — no migration was needed since the `email` column already exists and is nullable.

## How It Works

1. **Middleware** (`StudentInformationCheck`) runs on every student request and shares `$studentHasEmail` (boolean) with all views.
2. **Base layout** checks `auth()->user()->user_type === 'STUDENT' && !$studentHasEmail` — if true, renders a static-backdrop modal.
3. **Modal** auto-shows on page load with jQuery. The student cannot close it without entering an email.
4. **AJAX POST** to `student/update-email` validates the email and saves it. On success, a SweetAlert2 popup appears and the page reloads (modal won't reappear since email is now set).
5. **Validation errors** (invalid format, duplicate email) display inline under the input field.

## Manual Verification Steps

1. Log in as a student whose `users.email` is `NULL` → modal should appear and be un-closable
2. Submit invalid email (e.g. `notanemail`) → inline error shown
3. Submit valid unique email → SweetAlert2 success, page reloads, modal gone
4. Log in as a student who has an email → no modal appears
